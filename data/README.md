# Data layout

```
data/
├── raw/                  # Unmodified source files (HF dumps, Kaggle CSVs, etc.)
├── processed/            # Chunked/cleaned docs + persisted Qdrant index
└── knowledge_base/
    ├── it_docs/          # IT knowledge-base documents (.txt / .json)
    └── hr_docs/          # HR knowledge-base documents (.txt / .json)
```

## Sources (Option C — real + synthetic)

- **HR:** `strova-ai/hr-policies-qa-dataset` from Hugging Face (loaded by `ingestion.py`)
- **IT/HR:** Synthetic Q&A docs generated by `scripts/generate_synthetic_docs.py`
- **IT:** Add a Hugging Face IT dataset in `ingestion.py::load_huggingface_it()` once you find one

## Sizing target

Aim for roughly **30–50+ chunks per domain** — `build_knowledge_base.py` will warn you if you're below 30.

## Deliberately uncovered topics (escalation test cases)

These topics are intentionally NOT in the knowledge base so you can verify that `decide_escalation()` correctly routes them to ticket creation instead of guessing.

### IT gaps
- GDPR personal data deletion request process
- Provisioning a new AWS EC2 instance or cloud resource
- Kubernetes cluster access and kubectl setup

### HR gaps
- Parental leave and maternity / paternity entitlement details
- HIPAA compliance training and certification requirements
- Executive compensation, equity, and stock option vesting
